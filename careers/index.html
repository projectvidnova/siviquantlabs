// AFTER YOU DOWNLOAD

The path.

The zip contains your brief (PDF) and a sample dieline (.pdf + .png). Here’s exactly what to do.

01

Read the brief

Open SiviQuantLabs_Build_Challenge.pdf — it has the full spec, the dieline primer (cut vs. crease lines), and exactly what we grade.

02

Build it

A web app that lets you upload the dieline, builds a 3D box from it, and folds it closed with a clearly visible animation — then lets you orbit and rotate it. Use the included sample_dieline files as your input. Any frontend framework, any 3D library (Three.js, Babylon.js, your call).

03

Record ~60 seconds

First 30 sec: demo the upload, the fold-closed, and the rotate. Next 30 sec: walk through your code structure and the fold logic. Talk over it — we want to hear how you think.

04

Push your code

GitHub repo or a zip — somewhere we can open it.

05

Send it in

Email your video + code link to connect@siviquantlabs.com, subject line Build Challenge — <your name> — within 3 days of downloading.

⏱
TIME LIMIT — 3 DAYS

From the moment you download, the clock starts. You have 3 days to send your video + GitHub link to connect@siviquantlabs.com. Working + explainable wins — we’re not grading polish.
